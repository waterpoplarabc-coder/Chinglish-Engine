# 目标
把当前“偶发词汇替换”的 MVP，升级为：只要输入是英文，就能按强度（L1-L5）稳定地产生“中文思维的英语表达”（中式英语），并且可解释、可回归、可扩展到中式法语/西语。

# 现状问题定位
- 现在 rulepack 主要是“地道表达→中式表达”的词组替换：只有命中少数短语才会变；普通句子大概率不变。
- 结构规则是概率触发：即使命中模板也可能不触发，导致“输入=输出”。

# 改造原则（保证你要的“中文表述方式”）
- **强度越高，改写越必然**：L4-L5 默认至少触发 1~N 个结构/话语层规则（保证“发生变化”）。
- **先改话语结构，再改词汇与语法**：优先做“中文思维”特征（背景先行、因果链、过程强调、话题-评论），再做搭配直译与语法弱化。
- **可解释**：每一步输出前后 diff + 规则ID + 触发原因。
- **可回归**：引擎提供 deterministic 模式（测试/回归用），避免随机导致用例漂移。

# 引擎层面改造（核心）
## 1) 引入“分层规则类型”，不再只有 lexical_substitution
新增 rule type（都放进 rulepack，可配置）：
- `discourse_template`：话语模板（“先…然后…最后…”，“其实/所以/因为…所以…”，“首先说明…”）
- `reorder_adverbial`：时间/地点/方式前置
- `cause_effect_rewrite`：因果链中文化（Because… so…）
- `topic_comment`：话题-评论结构（“About X, …” / “For this part, …”）
- `redundancy_injection`：中文式冗余与重复（“very”，“a little”，“also”，“for users”）
- `tense_simplification`：弱化时态/一致性（受控、仅高强度）
- `article_preposition_noise`：冠词/介词可控错配（受控、仅L5）

## 2) 规则执行策略改造：保证 L4/L5 必改
- 新增执行参数：`force_change`（UI可勾选；L4/L5 默认 true）。
- 执行流程：
  1) L0-L1（话语/结构）先跑
  2) L2（搭配/词汇）再跑
  3) L3-L4（形态弱化/文化迁移）最后跑
  4) 若 `force_change=true` 且无规则命中：执行一个“安全兜底模板”（例如时间前置+过程强调），保证输出变化。
- 冲突处理：优先级 + 互斥组（比如同一句只允许一个“因果模板”），避免规则互相打架。

## 3) 最小语义保护（防止改坏意思）
不引入重型依存分析的前提下，先做工程上可落地的保护：
- 否定词（not/never/no）不得删除
- 数字/单位/专有名词不改写
- 句末标点保留
- 允许“风格变差”，但不允许把肯定变否定、把数字改错

# rulepack 数据层改造
## 1) 扩展 pack.json schema
- 让每条规则支持：`min_level/max_level`、`priority`、`mutual_exclusion_group`、`trigger`（regex/关键词/句式条件）、`transform`（模板参数）。
- 现有 182 条词汇替换保留作为 L2 子集。

## 2) 快速补齐“结构/话语规则库”
- 从你现有文档里的 L0-L1 思维模板表开始落地（背景先行、过程强调、主语弱化等）。
- 首批目标：让任意英文陈述句在 L4/L5 下几乎都能匹配至少一个结构规则。

# API 与 UI 改造

  - `force_change: bool`

  - `explain: bool`（已存在，增强为 diff 链路）
- UI：
  - 强度选择旁新增“强制改写”开关
  - 解释面板展示每步变换（before/after）而不仅是规则ID
  - 若输出未变化，给出“未命中任何规则”的明确提示（并建议提高强度/打开强制改写）
