## 0. 这份文档告诉我们的“中文表达方式”是什么
来自文档的关键点（我只提会直接变成算法的部分）：
- **基本语序**：主语 + 谓语 + 宾语（SVO）
- **修饰语前置**：
  - 定语（修饰名词）在名词前
  - 状语（时间/地点/方式等）在动词/形容词前
  - 补语在中心语后（结果/程度）
- **结构类型**：单句 / 复句 / 特殊结构（把字句、被字句、连动句、兼语句）
- **句式分类**：陈述/疑问/祈使/感叹
- **复句关系**：并列/承接/递进/选择/转折/因果/条件/假设（靠关联词体现）

## 1. 目标重定义（按你要的“先中文再直译”）
把英文输入做两步：
1) 先识别它在中文里会是什么结构（单句/复句/特殊结构）+ 找到中文成分（主语、谓语、宾语、定语、状语、补语）
2) 再按中文语序拼回英文（允许不地道，但要“像中文说话”）

## 2. 生成“不同骨架”的方式（从中文结构出发，不是一条规则走到底）
我们不只做一个骨架，而是按“中文结构类型”准备一组骨架模板；同一句话可以给 2~3 个候选骨架。

### 2.1 单句骨架（对应文档第1部分）
- 骨架A（最常见）：`主语 + [时间][地点][方式] + 谓语 + 宾语`
- 骨架B（更强调时间）：`[时间] + 主语 + [地点][方式] + 谓语 + 宾语`
- 骨架C（更强调地点）：`[地点] + 主语 + [时间][方式] + 谓语 + 宾语`

例子（你的截图）：
- 输入：`I'm planning to go to China tomorrow.`
  - 识别：主语=I，谓语核心=plan，动作=go，中国=地点宾语，tomorrow=时间状语
  - 输出候选：
    - A：`I am planning tomorrow go (to) China`
    - B：`Tomorrow I am planning go (to) China`
（level 决定是否保留 to/the/a 这些功能词）

### 2.2 特殊结构骨架（对应文档第2部分）
先不追求一次全做完，但骨架必须按文档分类扩展：
- 连动句（连续动作）：`主语 + V1 + V2 + ...`
- 兼语句（宾语兼后句主语）：`主语 + V1 + 兼语 + V2 + ...`
- 把/被（处置/被动）：在英文输入里用可识别的模式做近似映射（不要求完全等价，只要求骨架像中文）

### 2.3 复句骨架（对应文档第4部分）
按“关联词关系”建骨架：
- 因果：`因为…所以…`
- 条件：`只要…就…` / `如果…就…`
- 转折：`虽然…但是…`
- 承接：`先…然后…`

这些会变成：识别英文里的 because/if/although/then 等触发词 → 生成对应的中文式英文骨架。

## 3. 引擎实现（分层，保证条理）
### 3.1 结构识别层（Structure Classifier）
输入英文 → 判断属于：
- 短语（如 Good morning）
- 单句（SVO）
- 复句（因果/条件/转折/承接…）
- 特殊结构（连动/兼语/把/被…）

### 3.2 成分抽取层（Slots）
在每类结构里抽取槽位：
- S（主语）/ V（谓语核心）/ O（宾语）
- T（时间状语）/ P（地点状语）/ M（方式状语）
- C（补语：结果/程度，可后置）

### 3.3 骨架生成层（Skeleton Generator）
- 根据结构类型选择 2~3 个骨架模板生成候选
- 选择策略：默认自动选择（最符合中文），也支持 UI 手动选 A/B/C
- **“没必要改就不改”**：如果输入已经满足某个中文骨架，输出保持不变

## 4. 清理旧规则（你要求的“清理之前规则”怎么做）
原则：默认只保留“骨架重排 + 直译词汇”，把“口癖类”全部迁走。
- 默认移除/禁用：First/So/Actually/For this part/then we can continue 等 prefix/suffix 注入
- 默认保留：Open the light / People mountain people sea 这类直译词汇（可单独开关）
- 兜底方式改为：切换到另一个骨架模板，而不是加口癖强行变化

## 5. UI/解释（让你一眼看懂系统在按中文做事）
- UI 新增：
  - “结构类型”：短语/单句/复句/特殊结构
  - “骨架模板”：自动/骨架A/骨架B/骨架C
  - “直译词汇开关”
  - “口癖规则（legacy）开关”（默认关）
- 解释展示三块：
  - 成分抽取结果（S/V/O/T/P/M/C）
  - 选择的骨架模板
  - Before/After 重排链路
