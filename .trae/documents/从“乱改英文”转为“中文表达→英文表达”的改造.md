# 我理解你的方法（并同意）
你要的“中式英语”不是随机加口癖，也不是胡乱错英语；你给的可操作方法是：
1) 先得到中文语义/中文表达（至少是“中文词序骨架”）
2) 再把中文的字词按顺序直译回英文
3) 如果原英文已经和中文词序一致，那就不需要改写（输出可以相同）

你举的例子完全明确了目标：
- `Thank you` ⇔ `谢谢你`（词序一致）→ 不应该改成 `You thank`，应保持 `Thank you`
- `I want to go to China tomorrow.` 对应中文更像 `我 想 明天 去 中国` → 输出希望是 `I want tomorrow go China`（或保留 `to` 视强度）

# 具体改造计划（确认后我再动代码）

## 1) 增加“中文骨架（Chinese Order Skeleton）”中间层
引擎对英文输入做轻量“角色标注”，得到一个中文词序骨架（不是完整翻译，而是可控的骨架）：
- 角色：S(主语/我)、V1(意图/想 want)、T(时间/明天 tomorrow)、V2(动作/去 go)、P(地点/中国 China)、O(宾语/…)
- 骨架顺序固定为：`S V1 T V2 P O`（可按句型微调）

这一步相当于你说的“先翻译成中文”，但工程上用的是可控的骨架与词对齐，避免引入大模型/在线翻译依赖。

## 2) 生成“中式英语”：按骨架顺序重排英文词/短语
- 对于识别到的时间词（today/tomorrow/yesterday/具体日期等）：
  - 若时间在句尾且句子包含意图动词（want/plan/need/try 等），移动到意图动词后：
    - `I want to go to China tomorrow.` → `I want tomorrow go to China`（L3-L4）
    - `I want tomorrow go China`（L5：可选去掉介词/冠词等中文式省略）
- 若输入本来已经是中文语序（比如 `I want tomorrow go China`），则不改（输出可相同）。

## 3) 短语与固定表达：按“中文直译是否需要变形”处理
建立一个小型短语表（phrasebook），并按你的原则分两类：
- **词序一致：保持不变**
  - `Thank you`（谢谢你）
  - `Sorry`（对不起）
  - `Goodbye`（再见）
- **词序不一致：按中文语序重排**
  - `Good morning`（早上好：名词在前）→ `Morning good`
  - `Good night` → `Night good`

并且：短语模式下禁止加任何“First/So/Actually/For this part”类口癖。

## 4) UI / API：让“中文骨架→直译”的过程可见、可控
- API 增加（或重用）参数：
  - `mode: "zh_skeleton_literal"`（默认）
  - `level`：L3-L5 决定“省略 to/the/a”等中文式省略强度
- 返回解释 steps 中增加两类信息：
  - `skeleton`: 识别到的角色与顺序（例如 `I=我, want=想, tomorrow=明天, go=去, China=中国`）
  - `reorder_step`: before/after 的重排

UI 侧展示：
- “中文骨架”面板（可折叠）
- “重排链路”面板（Before/After）

## 5) 回归用例（用你的例子锁死方向）
新增/更新回归：
- `Good morning` → `Morning good`
- `Thank you` → `Thank you`（不改）
- `I want to go to China tomorrow.` →
  - L3/L4：`I want tomorrow go to China.`
  - L5：`I want tomorrow go China.`（若开启省略）
- 若输入已是 `I want tomorrow go China`：输出必须相同
